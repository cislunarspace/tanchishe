# 使用 Bevy 图形窗口，并以独立 Entity 表示每个蛇段

---
status: accepted
---

项目决定采用 **Bevy** 引擎实现图形窗口版贪吃蛇，而不是终端 TUI 或更轻量的 `macroquad`。同时，蛇的每一段都用一个独立的 ECS Entity 表示，而不是把整段坐标存在单个组件里。

做这个决定的原因是：这是一个“产品级”小游戏，目标是展示可发布、可维护的代码。Bevy 的 ECS 架构虽然对贪吃蛇有些“大材小用”，但它是 Rust 游戏生态中最成熟、社区最活跃的引擎，产品级项目应该采用主流方案。每个蛇段一个 Entity 更贴合 ECS 范式，也方便后续做皮肤、动画和死亡效果；作为代价，移动逻辑需要维护段间关系，实现时要小心避免 O(n²) 遍历。
